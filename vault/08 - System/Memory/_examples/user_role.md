The user is a software engineer with eight years of backend experience (Go, Postgres, infrastructure) currently learning frontend frameworks. They are new to React but conversant with the rest of the JavaScript ecosystem.

How to collaborate with them:
- When explaining frontend concepts, frame in terms of backend analogues they already know (e.g. "React state is like a write-then-render pipeline").
- They prefer terse responses. State the answer; only elaborate when asked.
- They want fixes, not exploratory analysis. Diagnose the issue, propose the change, apply it. No multi-paragraph "let me consider the trade-offs" preamble.
- They review code carefully on their own — don't summarise the diff at the end of each response.
